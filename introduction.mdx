AutoMapper is a .NET library that solves the tedious problem of writing code to map one object to another. Instead of hand-writing dozens of property assignments, you declare your mappings once and AutoMapper handles the rest — automatically matching properties by name convention, supporting complex scenarios like flattening, inheritance, and LINQ projection.

## Why AutoMapper?

Mapping code between layers — for example from domain objects to DTOs — is repetitive and error-prone. AutoMapper's convention-based approach means that if your destination type follows naming conventions (e.g., `CustomerName` maps from `Customer.Name`), no explicit configuration is needed.

## Quick example

```csharp getting-started.cs
// 1. Define your types
public class Order { public Customer Customer { get; set; } public decimal GetTotal() { ... } }
public class OrderDto { public string CustomerName { get; set; } public decimal Total { get; set; } }

// 2. Configure AutoMapper (once at startup)
services.AddAutoMapper(cfg => {
    cfg.LicenseKey = "<your-license-key>";
    cfg.CreateMap<Order, OrderDto>();
});

// 3. Inject and use IMapper
public class OrdersController {
    private readonly IMapper _mapper;
    public OrdersController(IMapper mapper) => _mapper = mapper;

    public OrderDto Get(int id) {
        var order = repository.GetById(id);
        return _mapper.Map<OrderDto>(order);
    }
}
```

<Steps>
  <Step title="Install the NuGet package">
    Add AutoMapper to your project via the .NET CLI or NuGet Package Manager.
  </Step>
  <Step title="Configure your mappings">
    Call `AddAutoMapper` in your startup code and provide a license key and your `Profile` classes.
  </Step>
  <Step title="Inject IMapper">
    Request `IMapper` via constructor injection in your services and controllers.
  </Step>
  <Step title="Map your objects">
    Call `mapper.Map<TDestination>(source)` to transform objects at runtime.
  </Step>
</Steps>
